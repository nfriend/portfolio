Inspirograph began as an entertaining and nostalgic way to familiarize myself with D3.js and TypeScript. It quickly grew to be one of the most ambitious personal projects I've attempted, and I'm quite proud of the result. Inspirograph forced me to get creative with the mechanics of the user interface; it was of the utmost importance that manipulating the gears was a tangible, physical, and effortless experience.

Generating the gears

The first challenge was generating the gears. While I could have simply created a series of .PNGs, one for each gear size, I wanted the flexibility to easily create new gear sizes in the future (and potentially let users create custom gears). Unfortunately, this meant programmatically assembling SVG command strings into an SVG path that described the gear's toothy perimiter. To accomplish this, I had to dust off some of the trigonometry I hadn't touched since 10th grade Geometry. The result was rather headache-inducing, but ultimately allowed for a great deal of flexibility down the road.

Working with paths

Under the surface, Inspirograph assembles its SVG definitions by concatenating string after string of command definitions that describe each angle and line of the gear. In order to avoid directly manipulating strings, I wrote a few simple utility objects to wrap the SVG creation process in a compile-time safety net. So instead of doing this:

var svg = 'M ' + options.radius + ' 0 '; for (var i = 0; i < options.toothCount; i++) { svg += 'A' + options.radius + ' ' + options.radius + ' ' + 0 + ' ' + 0 + ' ' + 1 + ' ' + newX + ' ' + newY; // etc.... } return svg;

... I was able to take advantage of TypeScript's static typing and write this much safer and less error-prone version:

var pathBuilder = new SVG.PathBuilder(), pathBuilder.addCommand(new SVG.MCommand(options.radius, 0)); for (var i = 0; i < options.toothCount; i++) { pathBuilder.addCommand(new SVG.ACommand(options.radius, options.radius, 0, false, true, newX, newY)); // etc.... } return pathBuilder.toString();

I was initially tempted to forgo this extra abstraction and just build the strings by hand, but the extra time it took to write this abstraction quckly paid off when I moved on to more complex SVG drawings (like the Ring Gear and the Beam).
